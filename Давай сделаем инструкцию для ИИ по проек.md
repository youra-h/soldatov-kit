Давай сделаем инструкцию для ИИ по проекту. Мне надо чтобы в следущий раз ИИ уже понимал мой проект и я каждый раз не объяснял ему все заново. Вот инструкция:
1. Проект это UI компоненты, под разные реактивные фреймворки (React, Vue, Angular и т.д.).
2. Я использую TypeScript для разработки компонентов, чтобы обеспечить типизацию и улучшить качество кода. У меня много интерфейсов и типов, которые описывают структуру данных и пропсы компонентов.
3. В текущем проекте я занимаюсь обкаткой всей системы, поэтому сейчас это общий проект разделенный на несколько состовляющих:
    - core, здесь находится в ся бизнес логика, поведение, события и т.д. Код написан на TS он отделен от слоя представления, поэтому здесь нет кода ни одного фреймворка
    - ui-vue, здесь код представления, как компонент выглядит, он написан на реактивном Vue 3 (текущая версия 3.5.32, но она меняется, я постепенно обновляю пакеты)
    - foundation, это общие внешние пакеты и связи, которые будут присущи всем фреймворкам, на данный момент здесь лежат импортирование tailwind 4, в папке themes дефолтная тема в виде цветовой палитры (цветовая палитра расширяться не может), так же там файл с описанием всех используемых на данный момент компонентами цветов COLOR-USAGE.md (данный файл должен расширяться по мере добавления новых компонентов), ИИ нужно обязательно это учитывать и не придумывать новые цвета и не расширять цветовую палитру, желательно, чтобы она была максимально сжата, но при этом компоненты не теряли вида и папка styles где сейчас есть лишь небольшие утилиты для tailwind, здесь так же папка может расширяться.
    -demo, здесь у нас демонстрация созданных компонентов и визуальное тестирования, двумя способами, через создание instance и через классическую передачу props в компоненты. В демо уже есть все присущие для всех компонентов постоянные, это описание и вывод всех свойств компонента в Properties, папка components где находятся сами компоненты они разделены на три файла Component.vue - передаются свойства, Instance.vue передается Instance, Slots.vue здесь любые использования слотов, а тка же просто вывод разнообразных свойств иконок, всего вчего угодно для общей демонстрации компонента. В папке playgrounds находятся файлы, которые объдиняют в себе Properties, Component, Instance, Slots
4. Подробнее, что такое core и как с ним работать:
    - Все компоненты наследуются от TEntity здесь самые базовые свойства и методы
    - Далее TComponentModel, это отделенная часть от TComponentView, чтобы в ней хранились лишь свойства модели, которые можно было бы наследовать в другие ветки развития
    - TComponentView это уже сам базовый визуальный компонент, это то на чем строятся визуальные копмопненты. Здесь полно свойств и событий
    - TStylable имеют визуальые свойства Variant, Size чтобы была возможность менять вид компонента
    - TControl все контролы наследуются от TStylable и дополнительно могут иметь свойства блокировки компонента и фокусировки
    - Так же есть ветка TInteractive, которая наследуюется TComponentView чтобы обойти мимо TStylable не имея возможности менять вид компонента, но при этом получая возможность блокировать его и фокусироваться
    - TTextable, TValueControl, TInputControl каждый из компонентов получает новые свойства, чтобы на их основе строить визуальные контролы с текстовыми полями, полями value или набором свойств input
    - TStateUnit - это обсобый класс на основе которго могут строиться некоторые свойства компонентов, чтобы они могли раширяться и полуать свобю уникальную логику при использовании компонентов будующими программистами. В папке states лежит много готовых стейтов, на основе которых уже есть свойства такие как visible, rendered и многие другие
    - TCollection, это набор компонентов в одной структуре с множеством методов для хранеия, добавления, поиска и т.д. TActivatableCollection расширяет TCollection и у него опоявляется свойство active, это значит что только один компонент в коллекции может быть активным. TSelectableCollection расширяет TCollection и у него появляется свойство selected, mode, апперируя этими свойствами мы может делать активным как только один элемент, тк и несколько, либо вообще запретить их делать активными.
    - в папке common лежит множество вспомогательных функций
    - в папке components лежат непосредственно компоненты, которые в UI станут визуальными
4. Есть еще папка icons в ней хзранятся текущие иконки проекта, такие как check, close, intermidiate и другие.
