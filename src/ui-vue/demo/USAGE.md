# Инструкция по работе с Playground

## Переключение между playground

Откройте файл [App.vue](App.vue) и измените переменную:

```typescript
const activePlayground = 'component-view' // измените на нужный ключ
```

Доступные значения указаны в объекте `playgrounds`.

## Запуск playground

Playground автоматически загружается при запуске приложения.

## Структура интерфейса

### 1. Панель свойств (Properties)
В верхней части находится панель управления свойствами компонента:
- **visible** - чекбокс для управления видимостью компонента
- **rendered** - чекбокс для управления рендерингом компонента
- **tag** - выпадающий список для выбора HTML-тега (div, span, section и т.д.)

### 2. Демонстрационные столбцы

#### Props Demo (слева)
- Компонент получает свойства напрямую через props
- Логирует события Vue компонента
- Синяя рамка для визуального отличия

#### Instance Demo (центр)
- Компонент работает с instance объектом
- Логирует события как от instance (core), так и от Vue компонента
- Зелёная рамка для визуального отличия
- Автоматически синхронизируется с изменениями в панели свойств

#### Slots Demo (справа)
- Статические примеры использования слотов
- Демонстрирует различные варианты контента:
  - Простой текст
  - Множественные элементы
  - Сложная вложенная структура

### 3. Event Log (внизу)
Лог событий отображает:
- **Временную метку** - точное время события с миллисекундами
- **Источник события**:
  - `props` (синий) - события из Props Demo
  - `instance` (зелёный) - события из Instance Demo  
  - `core` (фиолетовый) - события от core instance
  - `vue` (оранжевый) - события от Vue компонента
- **Название события** - тип произошедшего события
- **Payload** - данные события (если есть)

## Тестирование

1. Измените свойство в панели Properties
2. Наблюдайте, как оба компонента (Props Demo и Instance Demo) реагируют на изменения
3. Проверьте Event Log - увидите события от обоих источников
4. Slots Demo остаётся статичным, но также реагирует на изменения свойств

## Типичные сценарии тестирования

### Проверка видимости
1. Снимите галочку с `visible`
2. Компоненты должны скрыться
3. В логе увидите события: `beforeHide`, `hide`, `afterHide`, `change:visible`

### Проверка рендеринга
1. Снимите галочку с `rendered`
2. Компоненты полностью удалятся из DOM
3. В логе увидите события: `change:rendered`

### Проверка смены тега
1. Выберите другой тег из списка
2. Компоненты пересоздадутся с новым тегом
3. Можете проверить в DevTools, что тег действительно изменился

## Различия между Props Demo и Instance Demo

**Props Demo:**
- Только Vue события
- Управление через реактивные props
- Подходит для простых случаев

**Instance Demo:**
- События и от core instance, и от Vue компонента
- Прямое управление instance
- Полный контроль над состоянием
- Видны все внутренние события core логики
